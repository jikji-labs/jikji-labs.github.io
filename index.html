Jikji는 단일 정적 Go 바이너리로 실행되는 모듈형 에이전트 프레임워크입니다. 결정적(deterministic) 에이전트 루프, 영속적인 오케스트레이션 상태, 그리고 처음부터 내장된 거버넌스를 노트북에서 엔터프라이즈 클러스터까지 동일하게 제공합니다.

The Name이름의 유래

Named after the world's first movable metal type book 세계 최초의 금속활자본에서 따온 이름

Jikji (직지심체요절) was printed in Korea in 1377 — the oldest surviving book printed with movable metal type, decades before Gutenberg. Movable type changed publishing the same way Jikji aims to change agents: composable pieces, set precisely, producing reliable output at scale. 직지심체요절은 1377년 고려에서 간행된, 현존하는 세계에서 가장 오래된 금속활자본입니다. 구텐베르크보다 수십 년 앞섰죠. 금속활자가 출판을 바꾼 방식 그대로, Jikji는 에이전트를 바꾸려 합니다. 조합 가능한 부품을 정밀하게 조판해, 대규모에서도 신뢰할 수 있는 결과를 만들어내는 것.

Every module carries a letterpress name — the foundry casts the runtime, type faces the models, sorts are the tools, the compositor sets sessions, the forme locks the agent loop, and the press serves it to the world. Even the default port is 1377. 모든 모듈이 활판인쇄 용어로 이름 지어져 있습니다. foundry(주조소)는 런타임을 부트스트랩하고, type(활자)은 모델을, sorts(낱자)는 도구를, compositor(식자공)는 세션을, forme(판)은 에이전트 루프를, press(인쇄기)는 API를 담당합니다. 기본 포트마저 1377입니다.

Why Jikji왜 Jikji인가

What makes it different from other agent frameworks 다른 에이전트 프레임워크와 무엇이 다른가

Most agent frameworks are Python libraries glued to a vector DB, with opaque loops and governance bolted on later. Jikji is built like infrastructure from day one. 대부분의 에이전트 프레임워크는 벡터 DB에 붙인 파이썬 라이브러리이고, 루프는 불투명하며 거버넌스는 나중에 덧붙입니다. Jikji는 처음부터 인프라처럼 설계되었습니다.

01

One static binary, zero dependency hell단일 정적 바이너리, 의존성 지옥 없음

Written in Go and shipped as a single static binary. No Python environments, no dependency conflicts. The same binary runs on your laptop, as split services, or in Kubernetes with the included Helm chart. Go로 작성되어 단일 정적 바이너리로 배포됩니다. 파이썬 환경 구성도, 의존성 충돌도 없습니다. 같은 바이너리가 노트북에서도, 분리된 서비스로도, 포함된 Helm 차트로 Kubernetes에서도 그대로 동작합니다.

02

A deterministic, recoverable agent loop결정적이고 복구 가능한 에이전트 루프

The forme loop models every step as explicit actions and observations — with checkpointing and recovery built into the engine. Agent runs are replayable and auditable, not opaque chains that vanish on crash. forme 루프는 모든 단계를 명시적인 액션과 관찰(observation)로 모델링하며, 체크포인트와 복구가 엔진에 내장되어 있습니다. 에이전트 실행은 크래시와 함께 사라지는 블랙박스가 아니라, 재생(replay)과 감사가 가능한 기록입니다.

03

Governance built in, not bolted on덧붙인 게 아닌, 내장된 거버넌스

Multi-tenancy, scoped API keys, RBAC, structured audit trails with retention, policy gates on tool use, and human-in-the-loop approvals — all in the core, plus PII/secret redaction with Korean locale patterns. 멀티테넌시, 스코프 기반 API 키, RBAC, 보존 정책이 있는 구조화된 감사 로그, 도구 사용에 대한 정책 게이트, 휴먼 인 더 루프(HITL) 승인까지 모두 코어에 들어 있습니다. 한국어 로케일 패턴을 지원하는 개인정보·시크릿 마스킹도 기본입니다.

04

Scale-free: laptop → services → grid스케일 프리: 노트북 → 서비스 → 그리드

Local-first durable state on SQLite. Start in one process, split into services, then push tool execution to remote nodes over TLS relay tunnels — without changing your agents. SQLite 기반의 로컬 우선 영속 상태. 한 프로세스로 시작해 서비스로 분리하고, TLS 릴레이 터널을 통해 원격 노드로 도구 실행을 확장할 수 있습니다. 에이전트 코드는 그대로입니다.

05

Memory that actually learns실제로 학습하는 메모리

The plate cognitive memory plane extracts facts from conversations, consolidates them in background "dreaming" passes, and serves hybrid recall with context compaction. lexicon builds a knowledge graph from documents with contradiction detection. plate 인지 메모리 플레인은 대화에서 사실을 추출하고, 백그라운드 "드리밍" 과정으로 통합하며, 컨텍스트 압축과 하이브리드 리콜을 제공합니다. lexicon은 문서에서 지식 그래프를 구축하고 모순 탐지까지 수행합니다.

06

Open protocols, no lock-in오픈 프로토콜, 락인 없음

Serves an OpenAI-compatible API and speaks MCP (server and client), ACP, and A2A for agent-to-agent collaboration. Works with OpenAI-compatible, Anthropic-compatible, Ollama, and GLM/Z.AI backends. OpenAI 호환 API를 제공하고, MCP(서버·클라이언트), ACP, 에이전트 간 협업을 위한 A2A 프로토콜을 지원합니다. OpenAI 호환, Anthropic 호환, Ollama, GLM/Z.AI 백엔드와 함께 동작합니다.

A print shop of composable modules 조판실처럼 구성된 모듈들

Get Started시작하기

Running in under a minute 1분 안에 실행하기

# start the server (default port 1377 — yes, that 1377)서버 시작 (기본 포트 1377 — 네, 그 1377입니다)
go run ./cmd/jikji serve --config config.example.yaml

# check your local environment로컬 환경 점검
go run ./cmd/jikji doctor

# open the interactive chat TUI인터랙티브 채팅 TUI 열기
go run ./cmd/jikji chat --mode tui

# or call the OpenAI-compatible API또는 OpenAI 호환 API 호출
curl -X POST http://127.0.0.1:1377/v1/chat/completions \
-H 'Content-Type: application/json' \
-d '{"model":"jikji-echo","messages":[{"role":"user","content":"hello"}]}'
⚠ Jikji is under active development. APIs and module boundaries may change before the first stable release. This page describes what exists in the codebase today. ⚠ Jikji는 활발히 개발 중입니다. 첫 안정 릴리스 전까지 API와 모듈 경계는 변경될 수 있습니다. 이 페이지는 현재 코드베이스에 실제로 존재하는 내용을 기준으로 작성되었습니다.
